# PM Tool – Dashboard System Context

## 1. Project Objective
Build a centralized dashboard system to manage and monitor the entire PM Tool project.

The system should allow the PM team to:
- monitor rollout progress
- track system issues
- support decision-making
- manage multiple project phases in one place

## 2. Current Scope
At the current stage, the system includes 2 main modules:
1. Goal Setting Dashboard (Mass Roll-out)
2. Issue Tracking Dashboard

## 3. Future Vision
The dashboard system must be scalable to support additional modules in the future, such as:
- Performance Review tracking
- KPI / OKR monitoring
- System usage analytics
- User adoption tracking
- SLA monitoring

The system should not be designed as a one-time dashboard.
It should be designed as a modular dashboard platform that can expand over time.

## 4. Core Principles
- Single tool for the entire PM Tool project
- Modular design
- Easy to extend
- Action-oriented, not report-only
- Simple and maintainable

## 5. User Roles
- PM: monitor progress and drive execution
- HRBP: track department-level progress
- Admin: upload data and maintain the system
- Leadership: view high-level project health

## 6. Key Capabilities
The system should support:
- Upload raw data from Excel files
- Auto-generate dashboards
- Switch between modules easily
- Track progress, issues, and trends
- Support future module expansion

## 7. Design Direction
- Single system
- Multi-module structure
- Tab-based or toggle-based navigation
- Clean UI and high usability
