# PM Tool Dashboard Context

## 1. Project Background
PM Tool has entered the Mass Roll-out stage.
At this stage, the project team needs a simple dashboard to monitor Goal Setting execution progress.

The dashboard is for daily operation control, not for complex analytics.
It must help PM quickly identify:
- how many goal sheets have been set up
- which departments are delayed
- which approvers are overloaded
- which employees still need follow-up

## 2. Business Objective
Build a simple dashboard to support rollout tracking and daily follow-up during Goal Setting.

## 3. Actual Raw Data Columns
The current raw file contains these columns:
- Nhân viên
- Phòng ban
- Vị trí
- Tên phiếu mục tiêu
- Trạng thái
- Người duyệt

## 4. Key Dimensions
The dashboard should mainly use:
- Phòng ban
- Trạng thái
- Người duyệt
- Nhân viên

## 5. Key Business Questions
The dashboard should answer:
1. Total number of goal sheets
2. Number of completed goal sheets
3. Number of goal sheets not started
4. Number of goal sheets in progress
5. Completion rate by department
6. Pending workload by approver
7. Which employees still need follow-up

## 6. Guiding Principle
Keep the dashboard simple:
- single page
- easy to read
- easy to maintain
- no complex logic unless needed

## 7. Main Usage
- PM: track progress and push follow-up
- HRBP: track department progress
- Admin: upload raw data and refresh dashboard
